## 🟠 HIGH — Injection HTML dans `customMessage` des emails devis et facture

### Problème

`devis.sendEmail` et `factures.sendEmail` acceptent un champ `customMessage`
(message personnalisé de l'artisan) qui est injecté **brut** dans le HTML de
l'email sans aucune sanitisation :

```typescript
// server/routers.ts:875 (devis) et :1548 (factures)
body.replace('</body>',
  `<div ...>${input.customMessage.replace(/\n/g, '<br>')}</div></body>`)
```

La seule transformation est `\n → <br>`. Du HTML ou des attributs event handlers
(`<img onerror=...>`, `<a href="https://phishing.com">cliquez ici</a>`) passent
sans modification dans l'email envoyé au client.

### Risque

- Compte artisan compromis → emails de phishing vers les clients, branded Operioz.
- Artisan malveillant injecte des `<a>` redirigeant vers un site frauduleux.
- Certains clients webmail (Outlook, Thunderbird) exécutent du JS inline → XSS.

### Fix

Remplacer `customMessage` par du texte échappé avant insertion :

```typescript
function escapeHtml(s: string) {
  return s.replace(/[&<>"']/g, c =>
    ({ '&':'&amp;','<':'&lt;','>':'&gt;','"':'&quot;',"'":'&#39;' }[c]!));
}

const safeMsg = escapeHtml(input.customMessage).replace(/\n/g, '<br>');
body.replace('</body>', `<div ...>${safeMsg}</div></body>`)
```

À appliquer aux deux occurrences (`routers.ts:875` et `routers.ts:1548`).

## 🟡 MEDIUM — `buildPaymentConfirmedEmail` / `buildPaymentFailed` / `buildSubscriptionCanceled` jamais utilisés

### Problème

Trois builders d'email ont été écrits dans `emailService.ts` mais ne sont importés
nulle part :

- `buildPaymentConfirmedEmail` (l.412)
- `buildPaymentFailedEmail` (l.433)
- `buildSubscriptionCanceledEmail` (l.472)

Le `webhookHandler.ts` utilise son propre helper `subscriptionEmail()` local au lieu
de ces fonctions. Résultat : deux implémentations parallèles — si l'une est mise à
jour, l'autre ne l'est pas.

### Fix

Supprimer les trois builders non utilisés de `emailService.ts` **ou** refactoriser
le `webhookHandler` pour les utiliser. Pas de comportement cassé aujourd'hui, mais
risque de divergence à maintenir.
