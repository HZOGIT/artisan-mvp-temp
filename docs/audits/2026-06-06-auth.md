**Date** : 2026-06-06 · **Projet** : Lancement 30 juin

## 🔴 BLOCKER 1 — Signup ne crée pas l'artisan, la subscription ni les permissions

### Problème

Le flow `auth.signup` (`server/routers.ts:8910`) appelle `createUserWithPassword()`
puis retourne immédiatement. Il ne crée **pas** :
- le record `artisans` (table artisans, requis par 100 % des endpoints métier)
- le record `subscriptions` (requis pour le gate d'abonnement)
- les lignes `permissions_utilisateur` (requis pour tous les endpoints gated)

Résultat : un artisan qui s'inscrit via l'UI obtient un cookie valide mais l'app est
**totalement non-fonctionnelle** — chaque appel tRPC retourne soit une erreur 404
("Artisan non trouvé") soit FORBIDDEN (permissions vides).

### Preuve

- `server/routers.ts:8918` — `createUserWithPassword(input.email, input.password, input.name)`
  → seule opération DB du signup, ne touche que la table `users`.
- `server/_core/auth.ts:47-53` — `db.insert(users).values({…})` sans artisan/subscription/permissions.
- `server/routers.ts:576-577` — `getArtisanByUserId(ctx.user.id)` → `if (!artisan) return []`
  (toutes les queries métier silencieusement vides ou 404).

### Fix proposé

Dans la mutation `signup`, après `createUserWithPassword` :
1. `db.insert(artisans).values({ userId, nomEntreprise: name || email, email, specialite: '' })`
2. `db.insert(subscriptions).values({ artisan_id, plan:'trial', status:'trialing', trial_ends_at: +14j, … })`
3. Bulk insert `permissions_utilisateur` depuis `ROLE_TEMPLATES['artisan']` (même pattern que `seed-dev-user.sql`)

## 🔴 BLOCKER 2 — Aucun flow "mot de passe oublié"

### Problème

Il n'existe aucune route `forgotPassword` / `resetPassword` dans le codebase.
Un artisan qui oublie son mot de passe est **définitivement bloqué dehors** — aucun
moyen de récupérer l'accès sans intervention manuelle en base.

### Preuve

- `grep -rn "forgot\|resetPass\|reset_pass" server/` → 0 résultat.
- `client/src/pages/SignUp.tsx` existe mais aucun lien "mot de passe oublié" dans le
  flow de connexion.
- `server/routers.ts` (auth router, lignes 8910–9047) : uniquement `signup`, `signin`,
  `logout`, `updateEmail`, `updatePassword`, `deleteAccount`.

### Fix proposé

1. **Route `auth.forgotPassword`** (publicProcedure) : reçoit l'email, génère un token
   signé (JWT ou UUID stocké en DB avec TTL 1h), envoie un email via Resend avec lien
   `https://app.operioz.com/reset-password?token=…`.
2. **Route `auth.resetPassword`** (publicProcedure) : reçoit `token + newPassword`,
   vérifie validité + expiration, hash + update `users.password`.
3. **Page `/forgot-password`** et `/reset-password` côté client.
4. Lien "Mot de passe oublié ?" sur la page de connexion.
